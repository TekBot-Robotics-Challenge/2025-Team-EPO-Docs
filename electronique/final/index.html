1. Introduction

Dans le cadre du projet de tri intelligent de déchets à TEKBOT CITY, cette section détaille la conception de la partie électronique du système de convoyeur. Elle vise à permettre à toute personne, même novice, de comprendre la démarche, de la conception initiale jusqu'à l'implémentation finale du système embarqué.

Le système développé combine plusieurs technologies pour créer un convoyeur intelligent capable de détecter automatiquement la présence de déchets, d'analyser leur couleur et de transmettre ces informations via WiFi. Cette documentation couvre l'ensemble du processus de conception électronique, depuis le choix des composants jusqu'à la réalisation finale.

2. Objectifs Techniques

  • Détection de présence : Utilisation de capteurs LDR couplés à des lasers pour détecter l'arrivée et le positionnement des déchets.
  • Analyse colorimétrique : Identification précise des couleurs avec le capteur TCS3200.
  • Contrôle moteur : Pilotage du convoyeur via relais pour moteur DC.
  • Sécurité : Intégration d'un bouton d'arrêt d'urgence.
  • Communication sans fil : Transmission des données via ESP-01 et WiFi.
  • Autonomie : Système alimenté par batterie Lithium avec régulation stable.
  • Feedback visuel : LEDs indicatrices pour chaque couleur détectée.

Composant Rôle Justification
LDR + Laser Détection de présence/obstacle Faible coût, simple à interfacer

Arduino Nano Microcontrôleur principal (ATmega328P) Petit, puissant, supporté par KiCad et l'écosystème Arduino
AMS1117 / HW-131 Régulateurs 5V et 3.3V pour ESP et capteurs AMS1117 : régulation classique, HW-131 : régulateur plus stable

Code Arduino (Contrôleur principal)

Structure du firmware : 

{#define S2 2}              // TCS3200 - Sélection couleur
{#define S3 3}              // TCS3200 - Sélection couleur
{#define OUT 4}             // TCS3200 - Sortie PWM
{#define LDR_ENTREE A0}     // Détection présence entrée
{#define LDR_COULEUR A1}    // Détection zone couleur
{#define EMERGENCY_BTN 9}   // Bouton d'urgence
{#define LASER_1 11}        // Laser détection entrée
{#define LASER_2 12}        // Laser détection couleur
{#define RELAY_PIN 5}       // Commande relais moteur
{#define LED_ROUGE 6}       // LED indicatrice rouge
{#define LED_BLEU 7}        // LED indicatrice bleue
{#define LED_VERT 8}        // LED indicatrice verte
{#define LED_JAUNE 10}      // LED indicatrice jaune

Fonctions principales :

  1. Détection de présence : bool presence = analogRead(LDR_ENTREE) < 800;
  2. Détection colorimétrique :
String detecterCouleur() {
  while (true) {
    lireCouleur();
    if (red < 11 && green > 15 && blue > 15) return "rouge";
    if (red > 15 && green > 15 && blue < 11) return "bleu";
    if (red < 11 && blue < 11 && green < 11) return "vert";
    if (red < 10 && blue > 10 && green < 11) return "jaune";
    delay(100);
  }
}
  1. Contrôle moteur :
void demarrerConvoyeur() {
  moteurEnMarche = true;
  digitalWrite(RELAY_PIN, HIGH);
}

void stopConvoyeur() {
  moteurEnMarche = false;
  digitalWrite(RELAY_PIN, LOW);
}

void envoyerDonnees(const String& couleur) {
  HTTPClient http;
  http.begin(client, apiUrl);

  int httpCode = http.POST(payload);
}

9. Considérations Techniques

Calibrage des Capteurs

LDR (Light Dependent Resistor) :

  • Seuil de détection : 800 (valeur analogique).
  • Résistance de pull-up : 10kΩ.

TCS3200 (Capteur RGB) :

  • Fréquence de sortie : 100% (S0=HIGH, S1=HIGH).
  • Sélection couleur via S2/S3.

Gestion des Interférences

  • Temporisation entre mesures.
  • Filtrage matériel : Condensateurs de découplage sur alimentation.
  • Blindage des câbles capteurs.
  • Séparation physique de l'alimentation du moteur et de la carte.

10. Tests et Validation

Tests Unitaires

  • Test alimentation : Mesure tensions 5V/3.3V, vérification stabilité sous charge.
  • Test capteurs : Étalonnage LDR avec objets test, validation TCS3200 avec palette couleurs.
  • Test communication : Liaison série Arduino-ESP01, transmission WiFi.

Tests d'Intégration

  • Séquence complète : Détection → Analyse → Transmission.
  • Temps de réponse : 1 seconde < Temps de réponse < 5 secondes.
  • Fiabilité : > 95%.

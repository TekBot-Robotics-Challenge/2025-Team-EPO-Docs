1. Introduction

1.1 Présentation du Projet

La Plateforme Web Convoyeur Intelligent de Tri est un système IoT innovant développé dans le cadre du TRC 2025 par l'équipe TEKBOT. Cette solution combine hardware embarqué et interface web moderne pour automatiser et optimiser le processus de tri des déchets par couleur.

1.2 Vision du Projet

Créer une solution complète de tri intelligent qui automatise la détection et classification des déchets, visualise en temps réel les performances du système, guide les opérateurs avec des consignes vocales, et analyse les données pour optimiser les processus.

2. Contexte et Objectifs

2.1 Problématique

Le tri manuel des déchets présente plusieurs défis : erreurs humaines dans la classification, lenteur du processus de tri, manque de traçabilité des performances, et absence de données pour l'optimisation.

2.2 Objectifs Principaux

2.2.1 Objectifs Techniques

  • Développer un système de détection automatique par couleur.
  • Créer une interface web temps réel responsive.
  • Implémenter une API robuste pour la communication IoT.
  • Assurer une latence faible pour les détections.

2.2.2 Objectifs Fonctionnels

  • Tri automatique de 4 couleurs (jaune, bleu, vert, rouge).
  • Synthèse vocale pour guidage opérateur.
  • Tableau de bord analytique avec graphiques.
  • Historique des détections avec horodatage.

2.2.3 Objectifs Qualité

  • Interface utilisateur intuitive et moderne.
  • Système fiable avec gestion d'erreurs.
  • Performance optimisée pour l'IoT.
  • Code maintenable et documenté.

3. Analyse des Besoins

3.1 Besoins Fonctionnels

3.1.1 Détection et Classification

  • Détecter la présence d'un déchet sur le convoyeur.
  • Identifier la couleur du déchet (4 couleurs supportées).
  • Transmettre les données via Wi-Fi à l'API.
  • Horodater chaque détection avec précision.

3.1.2 Interface Utilisateur

  • Afficher l'animation du convoyeur en temps réel.
  • Mettre à jour les compteurs par couleur.
  • Fournir des consignes vocales automatiques.
  • Présenter un historique des détections.

3.1.3 Analyse et Reporting

  • Générer des graphiques de performance.
  • Calculer des métriques (débit, moyennes, variations).
  • Exporter les données pour analyse externe.
  • Alertes en cas de dysfonctionnement.

3.2 Besoins Non-Fonctionnels

3.2.1 Performance

  • Latence faible pour les détections.
  • Interface fluide .

3.2.2 Utilisabilité

  • Interface responsive (mobile/desktop).

4. Choix Technologiques

4.1 Architecture Matérielle

4.1.1 Microcontrôleur Principal

Arduino Nano (ATmega328P) : Faible coût et grande disponibilité, écosystème riche de capteurs compatibles, consommation énergétique optimisée, documentation extensive.

4.1.2 Module de Communication

ESP-01 (ESP8266) : Wi-Fi intégré avec stack TCP/IP, coût très compétitif, faible consommation en mode veille, compatible avec Arduino IDE.

4.1.3 Capteurs

Capteur de Couleur TCS3200 : Détection RGB précise, interface digitale simple, calibration programmable, coût abordable.

Capteur LDR couplé au laser Laser pour la presence : Détection précise sans contact, réponse rapide < 1ms, .

4.2 Stack Technologique Logicielle

4.2.1 Frontend Web

React 18 + TypeScript : Écosystème mature et performant, typage statique pour la robustesse, hooks pour la gestion d'état moderne, large communauté et support.

4.2.2 Backend API

Node.js + Express.js : Performance élevée pour l'IoT, écosystème npm riche, JavaScript full-stack, gestion native des WebSockets.

4.2.3 Outils de Développement

Vite : Build ultra-rapide avec ESBuild, Hot Module Replacement instantané, configuration minimale, support TypeScript natif.

Recharts : Graphiques React natifs, performance optimisée, customisation complète, API intuitive.

Protocoles Utilisés : UART (Arduino ↔ ESP-01), Wi-Fi 802.11n (ESP-01 ↔ Réseau local), HTTP/REST (ESP-01 ↔ API Backend).

│   └── GET /status        # État système
├── Middleware
│   ├── CORS               # Cross-origin requests

6. Conception et Design

6.1 Principes de Design

La conception de l'interface utilisateur et l'expérience utilisateur ont été guidées par des principes de design centrés sur l'utilisateur, en mettant l'accent sur la simplicité, l'efficacité et l'esthétique.

6.1.1 Design System

Nous avons établi un système de design cohérent pour garantir une expérience utilisateur uniforme à travers toutes les plateformes.

  • Palette de Couleurs : Bleu industriel (#3B82F6), Gris moderne (#6B7280), Vert succès (#22C55E), Rouge attention (#EF4444).
  • Couleurs des déchets : Jaune (#F59E0B), Bleu (#3B82F6), Vert (#10B981), Rouge (#EF4444), Noir (#111827), Blanc (#FFFFFF).

6.1.2 Typographie

La typographie joue un rôle crucial dans la lisibilité et l'expérience utilisateur globale.

  • Famille de polices : Inter (système), avec une police de secours sans-serif.
  • Échelle de tailles : 12px, 14px, 16px, 18px, 24px, 32px.
  • Poids : Regular (400), Medium (500), Bold (700).
  • Interlignage : 1.5 pour le corps de texte, 1.2 pour les titres.

6.1.3 Système de Spacing

Un système de spacing cohérent est utilisé pour maintenir l'alignement et l'équilibre visuel.

  • Échelle basée sur 8px : xs (4px), sm (8px), md (16px), lg (24px), xl (32px), 2xl (48px).

6.2.2 Composants Clés

Les composants clés de l'interface utilisateur sont conçus pour offrir une expérience utilisateur fluide et intuitive.

Animation Convoyeur
  • Dimensions : 100% de la largeur, 128px de hauteur.
  • Texture : Dégradé métallique avec lignes de guidage.
  • Items : Cubes 3D colorés avec ombres.
  • Zones : Détection (rouge) et Tri (vert) marquées.
  • Animation : 60 FPS, mouvement fluide de gauche à droite.
Compteurs de Déchets
  • Layout : Grille 2x3 (mobile) → 3x2 (desktop).
  • Cartes : Bordure colorée, ombre subtile.
  • Contenu : Pastille couleur + nom + nombre + barre de progression.
  • Animation : Transition smooth sur les changements.
Graphiques Analytiques
  • Types : Barres, Camembert, Courbe temporelle.
  • Couleurs : Cohérentes avec les déchets.
  • Interactivité : Tooltips, zoom, sélection.
  • Responsive : Adaptation automatique mobile/desktop.

6.3 Expérience Utilisateur (UX)

L'expérience utilisateur a été conçue pour être intuitive et facile à utiliser, en mettant l'accent sur la simplicité et l'efficacité.

6.3.1 Parcours Utilisateur Principal

  1. Arrivée : Statut système visible immédiatement.
  2. Observation : Animation convoyeur attire l'attention.
  3. Compréhension : Compteurs montrent la répartition.
  4. Analyse : Onglet graphiques pour approfondir.
  5. Contrôle : Panneau pour actions (pause, simulation).

6.3.2 Feedback Utilisateur

  • Visuel : Animations, changements de couleur, badges.
  • Auditif : Synthèse vocale pour consignes.
  • Temporel : Timestamps précis, durées d'animation.

6.4 Design Responsive

Le design responsive assure que l'interface utilisateur est accessible et utilisable sur une variété de dispositifs et de tailles d'écran.

6.4.1 Breakpoints

  • Mobile : 320px - 767px.
  • Tablette : 768px - 1023px.
  • Desktop : 1024px+.
  • Large : 1440px+.

6.4.2 Adaptations par Écran

  • Mobile (320px+) : Navigation par onglets, Convoyeur adapté en hauteur, Compteurs en grille 2x3, Graphiques simplifiés, Contrôles en bottom sheet.
  • Tablette (768px+) : Sidebar partiellement visible, Convoyeur taille intermédiaire, Compteurs en grille 3x2, Graphiques complets, Contrôles en sidebar.
  • Desktop (1024px+) : Layout complet 3 colonnes, Convoyeur pleine largeur, Tous les composants visibles, Graphiques interactifs avancés, Contrôles intégrés.
